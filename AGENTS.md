## Project Notes

- The project already has a `.resx`-based localization system, but Simplified Chinese coverage is incomplete.
- Several `zh-Hans` resource files exist only as empty placeholders, so the UI silently falls back to English.
- A noticeable amount of user-facing text is still hard-coded directly inside `*.cs` config UI files instead of being routed through localization resources.
- This repository depends on git submodules (`ECommons`, `PunishLib`, `WrathCombo.API`). If those folders are present but empty/uninitialized, the solution will look structurally complete but will not build.
- The current project file targets `net10.0-windows`. Building requires a .NET SDK that supports .NET 10; using only .NET 8 will fail immediately with `NETSDK1045`.
- When adding or changing UI text, prefer updating/adding `.resx` entries first and only hard-code strings as a last resort.
- Before claiming the app is fully localized, check both:
  - existing `zh-Hans` resource coverage
  - hard-coded `Draw*("...")` strings inside config/UI code
- This repository is already connected to `SumomoAkihime/WrathCombo`. Keep that remote unless there is an explicit migration, because the sibling project in this workspace uses a different GitHub namespace and it is easy to mix them up.
- Plugin packaging is now wired through `.github/workflows/publish-plugin.yml`. Because the project targets `.NET 10` and depends on Dalamud files plus git submodules, the most reliable way to generate a fresh `latest.zip` in this workspace is GitHub Actions, not the current local machine.
- If the custom plugin repo needs a new `WrathCombo` package, trigger or wait for the GitHub Actions packaging workflow, then use the produced `latest.zip` artifact as the source of truth. Do not invent repository metadata from the `.csproj` alone when the packaged zip already exists.
- The workflow now produces both the upstream-compatible standard package and a `WrathComboCN` variant package. Use the `CN` variant for the custom repo when you need it to coexist with the upstream `WrathCombo` entry in Dalamud.
- `WrathCombo/Resources/Localization/Presets/CustomComboPresets.zh-Hans.resx` was effectively empty, which causes the feature list pages to fall back to English for most preset names and descriptions. Regenerate or update that file when syncing upstream preset changes.
- Many localized `.resx` files here are UTF-8 without BOM. Reading them through Windows PowerShell's default ANSI path can show fake mojibake and even false XML parse failures. When inspecting or scripting them, force UTF-8 with `[System.IO.File]::ReadAllText(..., [System.Text.Encoding]::UTF8)`.
- `Resources/Localization/UI/Settings/SettingsUI.zh-Hans.resx` and `SettingsCfgUI.zh-Hans.resx` were previously swapped: the page-level strings lived in the config-key file and the config-key strings lived in the page-level file. When Settings suddenly falls back to English while other tabs still translate, check for key/file mismatches like this before touching runtime culture code.
- `ff14.huijiwiki.com` action pages sit behind Cloudflare. Direct `Invoke-WebRequest` and plain `requests` calls usually return the challenge page. Use `tools/Build-HuijiActionMap.py` with `cloudscraper` against `api.php?action=parse&page=Action:<id>` instead.
- Huiji action pages expose the JP+EN names inside a single `<font>` block. If you split on the first ASCII-looking token, multi-word names like `Hard Slash` and `Lucid Dreaming` get truncated and the zh-Hans preset replacements silently miss them. Prefer the current parser plus symbol-derived aliases from `HardSlash -> Hard Slash`.
- `tools/Update-CustomComboPresets-zhHans.ps1` currently translates preset values line-by-line after splitting on newlines. Multi-line regexes against the original full paragraph will never match there; add per-line rules or refactor the split stage first.
- `WrathComboCN` versioning should advance on every release. The packaging workflow now derives the CN package `AssemblyVersion` from the upstream four-part version plus the GitHub Actions run number, instead of a fixed suffix.
